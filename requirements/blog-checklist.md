# Blog Checklist

## Content
- [ ] Title is clear and compelling
- [ ] Opening paragraph hooks the reader
- [ ] Each section has a clear purpose
- [ ] Claims are supported by evidence or sources
- [ ] No unsupported factual claims
- [ ] Conclusion summarizes and provides next steps or call to action
- [ ] TODOs from draft are resolved

## Structure
- [ ] Heading hierarchy is logical (H1 → H2 → H3)
- [ ] Paragraphs are short (2-4 sentences)
- [ ] Bullet lists used where appropriate
- [ ] Code blocks formatted with language tags
- [ ] Links use descriptive anchor text

## Style
- [ ] Tone matches the selected writing style
- [ ] No jargon without explanation
- [ ] Sentences are concise and readable
- [ ] Active voice preferred over passive
- [ ] Consistent tense throughout
- [ ] No clichés or filler phrases

## Technical
- [ ] All code examples are correct
- [ ] Commands are tested and reproducible
- [ ] Dependencies or prerequisites are listed
- [ ] No broken internal links
- [ ] Images have alt text
