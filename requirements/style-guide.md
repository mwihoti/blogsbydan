# Style Guide

## Voice and Tone
- Write as a knowledgeable peer, not an academic or a salesman
- Use "you" to address the reader directly
- First-person ("I", "we") is fine for personal essays and build-in-public posts
- Avoid corporate jargon, buzzwords, and hedging language
- Be honest about uncertainty — use "I think" or "in my experience" rather than definitive claims

## Structure
- Start with the reader's problem or question
- Front-load each section with the key takeaway
- One idea per paragraph
- Use transitions between sections ("Now that we've covered X, let's look at Y")
- End with a clear takeaway or call to action

## Formatting
- `inline code` for commands, functions, file paths, and short code references
- Fenced code blocks with language for longer code examples
- `**bold**` for emphasis (sparingly)
- `*italic*` for book titles, foreign words, or definitions
- `---` for horizontal rules between major sections (sparingly)
- Blockquotes (`>`) for pull quotes or callouts

## Writing Level
- Target a reader who is smart but not necessarily familiar with the topic
- Define acronyms on first use
- Keep sentences under 25 words where possible
- Use examples to illustrate abstract concepts

## What to Avoid
- Excessive adverbs and adjectives
- Passive voice when active is clearer
- Unsupported claims or speculation presented as fact
- Walls of text without paragraph breaks
- Inside jokes or references the reader won't understand
